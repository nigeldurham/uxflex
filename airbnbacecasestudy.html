Project Overview

Our team of 4 designers created a
marketplace for events and spaces.

My contributions included :

   Interface Design
   Brand Design
   Presentation Design

96%  of users "enjoyed the experience"

Personas

Aaron

Aaron is sharing an event space on airbnb.

He is a young millenial who enjoys helping others, actively seeking new connections, and creating meaningful experiences.

Beth

Beth is looking for a place to host her event.

She is a responsible, confident professional who follows a strict schedule.

Problem Statement

Event hosts are seeking spaces to host their events.

How might we connect available spaces with events seamlessly on airbnb?

Solution

We wireframed a solution that allows events and spaces to exist in tandem on the platform - users can find one another via a central events hub, Ace Events.

Tasks

In testing, users will be tasked with either:

Posting a space to be discovered ( Aaron's flow )

Or posting an event, then searching for a space for that event  ( Beth's flow )

Following are the primary pain points for users across task flows and prototype iterations.

Iteration 1

For our primary iteration, both user tracks noted that some CTAs were in uncommon locations or used unexpected terminology.

for example

Landing page search bar location was unexpected.

Details page event tags were obscure.

Iteration 2

Introducing host and guest modes for spaces and events.

User profile ( L ) and event detail confirmation ( R ) screens finalized.

Iteration 2

In our second iteration, we introduced a "mode" feature where relevant paths (space ("host") or event ("guest")) could be selected from the primary landing page. Users were confused by the term "mode."

for example

Landing page CTA "mode" was unclear for the task flow.

Landing page search bar suggestions were confusing.

Final Iteration

High fidelity prototype with design modifications following two rounds of usability testing.

Notes refer to the most impactful modifications.

Iteration 3.1

Situating our product within the airbnb flow, users were encouraged to explore from brand-landing to task completion.

for example

Introduced product branding in promo slot on the airbnb app.

Users are invited to click to explore product.

Iteration 3.2

Both user-tracks are able to add multiple events or listings, managing them via a management tab.

for example

  Added ability to manage listings.

Iteration 3.3

Social accounts added for easy sharing and promotion.

for example

Messaging unlocked following input of event details.

Social accounts can be added for seamless sharing and promotion.

Iteration 3.4

Exploration is key - users can search or browse events and listings by location.

for example

Infinite scroll added on search page for open browsing of all local listings.

Take Homes

Usability testing indicated that some cta locations or terminologies were confusing.

By simplifying language and reorganizing assets or ctas, our experience better aligned with user expectations.

Results

96% of users enjoyed app flow

100% completed task goals

Brand-familiar users commented that our product fit seamlessly within the airbnb experience.
